## 🎯 Application Overview

**EVA DevBench** is a production-quality web application designed for AI-assisted software engineering work in secure government environments. It orchestrates complex work packages (reverse engineering, scaffolding, refactoring, testing, documentation) and produces auditable artifacts through secure, traceable runs.

### Target Users
- Government software engineers working with legacy codebases (COBOL, client-server, three-tier architectures)
- Technical debt reduction teams
- Code reviewers requiring AICOE-grade governance
- Engineering teams modernizing 60+ years of legacy applications

## 🏗️ Architecture

### Technology Stack
- **Framework**: React 19.2.0 with TypeScript 5.7.3
- **Build Tool**: Vite 7.2.6
- **Styling**: Tailwind CSS 4.1.17 + tw-animate-css
- **UI Components**: Shadcn/ui v4 (40+ pre-installed components)
- **Icons**: Phosphor Icons React 2.1.10
- **State Management**: React hooks + useKV (persistent state)
- **HTTP Client**: Fetch API with custom APIM integration
- **Real-time**: Server-Sent Events (SSE) for run streaming
- **Notifications**: Sonner 2.0.7
- **Forms**: React Hook Form 7.67.0 + Zod 3.25.76
- **Animations**: Framer Motion 12.23.25

### Key Features Implemented

#### 1. **Project Management**
- Create and manage projects with metadata (name, description, cost center, owners)
- Set active project context
- Recent projects tracking

#### 2. **Context Bundle Builder**
- Create secure context bundles with sensitivity levels (ULL/PB)
- Upload files via signed URLs with progress tracking
- Add text constraints for AI guidance
- Finalize bundles with cryptographic hashing
- Immutable sealed bundles with audit trail

#### 3. **Work Package Catalog**
- Pre-defined work packages: reverse engineer, scaffold, refactor, tests, docs, learn
- Parameter configuration for each package type
- Purpose and data class selection

#### 4. **Live Run Console**
- Real-time SSE event streaming
- Agent timeline visualization
- Progress tracking with phase indicators
- Cost tracking (tokens in/out, estimated CAD)
- Run cancellation capability
- Warnings and errors panel

#### 5. **Enhanced Artifacts Viewer with Code Review**
This is the most sophisticated feature, implementing enterprise-grade code review capabilities:

##### **Review Mode Overlay**
- Toggle to enable guided code review interface
- Visual risk indicators: 🟢 Low (docs, tests) | 🟡 Medium (scaffolding) | 🔴 High (logic, auth)
- Large diff warnings (>100 lines changed)
- Core layer indicators (auth/, security/, domain/ files)
- Review checklist with persistent state:
  - Business logic inferred? (Yes/No/Unclear)
  - SME validation required?
  - Tests generated?
  - Security reviewed?
  - Performance considered?

##### **Enhanced Diff Viewer**
- **Syntax Highlighting**: TypeScript, JavaScript, Python, Java, C#, HTML, CSS, JSON, SQL, Ruby, Go, Rust, PHP, Kotlin, Swift
- **View Modes**: 
  - Unified inline view (default)
  - Side-by-side split view
- **Word-Level Diff**: Character-by-character highlighting within modified lines (toggleable)
- **File Tree Navigation**: Expandable/collapsible file structure
- **Line Numbers**: Original and modified line tracking

##### **Agent Rationale Per File** (Critical Feature)
Expandable "Why this change?" panel for each file showing:
- Agent name (Scaffolder, Refactorer, Tester, etc.)
- Input signals used (files, constraints, context)
- Explicit assumptions made
- Confidence level (Low/Medium/High)
- Natural language reasoning

**Purpose**: Addresses "code feels junior-level" feedback by making AI intent transparent

##### **File Provenance & Traceability**
Complete audit trail badge on every file:
- Generated by: Agent name and version
- Model profile: AI model used (gpt-4o, gpt-4o-mini)
- Context bundle hash: Full SHA-256 with first 16 chars visible
- Run ID: Clickable ULID with copy-to-clipboard
- Timestamp: ISO 8601 with local formatting

**Benefit**: Every line auditable; supports AICOE governance requirements

##### **Semantic Grouping**
Files organized by architectural layer using heuristics:
- **Controllers**: API endpoints, route handlers
- **DTOs**: Data transfer objects, models, schemas
- **Services**: Business logic, repositories
- **Tests**: Test files, specs
- **Docs**: Markdown, README files
- **Config**: Configuration files

**Benefit**: Review by architectural layer instead of alphabetically

##### **"What Was NOT Touched" Panel**
Dedicated view showing:
- **Protected Zones**: Files AI cannot modify (auth/, security/, policy/)
- **Excluded by Constraints**: Files filtered by user rules
- **Intentionally Skipped**: Files detected but skipped with reasoning

**Purpose**: "The AI didn't roam freely—it stayed in its lane"

##### **File Review Status Markers**
Persistent review status tracking per file:
- ✅ **OK**: Approved, ready to merge
- 🧠 **Needs SME**: Requires subject matter expert
- 🔁 **Needs Re-run**: Should regenerate with new constraints
- ⏳ **Pending**: Not yet reviewed (default)

Features:
- Click-to-change dropdown on each file
- Review summary dashboard with progress tracking
- Bulk actions to mark all files at once
- Visual progress indicator (% reviewed)
- Persistent state via useKV (survives page refresh)

##### **"Explain This Hunk" Feature**
AI-powered contextual explanation for any diff section:
- What changed
- Why necessary
- What to watch out for
- How to test it

Uses Spark LLM API with context bundle grounding

##### **Change Risk Indicators**
Heuristic-based risk calculation per file:
- 🟢 **Low**: Comments, docs, tests, config
- 🟡 **Medium**: Scaffolding, moderate complexity (50-100 lines)
- 🔴 **High**: Branching logic, auth patterns, destructive ops, >100 lines, core layers

#### 6. **Audit & Evidence Trail**
- Run metadata (who, when, purpose, data class, trace ID)
- Token counts and cost estimation
- Models used per agent
- Complete warning/error log
- Export capability (JSON report)

#### 7. **Bilingual Support**
- Toggle between English and French
- Translation dictionary architecture
- Persistent language preference

#### 8. **Settings & Configuration**
- APIM base URL override for testing
- Demo mode toggle (mock data)
- Health check integration
- Language selection

#### 9. **Security Features**
- No secrets in browser
- All API calls through APIM gateway
- Required headers: x-evadb-project-id, x-evadb-trace-id, x-evadb-purpose, x-evadb-data-class
- Bearer token authentication (stub ready for Entra ID)
- Sensitivity warnings for PB data class

## 🔌 API Integration

### APIM Endpoints
Base URL: Configurable via settings (default: `/devbench/v1`)

**Implemented Endpoints**:
- `GET /health` - Health check
- `GET /capabilities` - Platform capabilities
- `GET /projects` - List projects
- `POST /projects` - Create project
- `POST /context-bundles` - Create bundle + upload plan
- `GET /context-bundles/{id}` - Get bundle details
- `POST /context-bundles/{id}:finalize` - Seal bundle
- `GET /work-packages` - List work package templates
- `POST /runs` - Start run
- `GET /runs/{runId}` - Get run status
- `POST /runs/{runId}:cancel` - Cancel run
- `GET /runs/{runId}/events` - SSE event stream
- `GET /runs/{runId}/artifacts` - List artifacts
- `GET /artifacts/{artifactId}` - Get artifact metadata
- `GET /artifacts/{artifactId}/content` - Download artifact

### Required Headers
```typescript
{
  'Authorization': `Bearer ${token}`,
  'x-evadb-project-id': projectId,
  'x-evadb-trace-id': traceId, // UUID v4
  'x-evadb-purpose': 'reverse_engineer' | 'scaffold' | 'refactor' | 'tests' | 'docs' | 'learn',
  'x-evadb-data-class': 'ULL' | 'PB'
}
```
